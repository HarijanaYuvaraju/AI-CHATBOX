    Learn the math
    Entropy in bits is estimated as E = L × log2(N), where L is password length and N is the size of the character set actually used (lowercase ≈26, uppercase ≈26, digits ≈10, symbols ≈32). More bits means an attacker guessing offline has to try more combinations before finding a match.

    Crack time here assumes an attacker can try 10 billion guesses per second against a fast, unsalted hash — a realistic worst case for a leaked database, not for a login form with rate limiting. Repeated characters, sequences, and known leaked passwords are penalized heavily, since attackers try those first.
